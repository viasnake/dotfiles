# Frontend Design

Use this skill when building or refining web components, pages, or applications that should feel intentionally designed rather than generic.

## Design approach

Before coding, choose a strong visual direction based on the product, audience, and constraints.

- Pick a specific tone and commit to it
- Make the design memorable through typography, layout, color, and motion
- Match implementation complexity to the visual idea
- Preserve the repository's existing design system when one already exists

## Frontend guidance

- Choose expressive, appropriate fonts instead of default-looking stacks
- Build a cohesive palette with clear contrast and well-defined accents
- Use CSS variables to keep the visual system consistent
- Prefer intentional motion such as page-load reveals, staggered entrances, or meaningful hover states
- Use backgrounds, shapes, textures, gradients, or patterns to create atmosphere
- Avoid generic AI aesthetics such as purple-on-white defaults, bland layouts, and predictable component compositions

## Output expectations

- Produce working production-grade code suited to the current stack
- Keep desktop and mobile behavior both in mind
- Balance creativity with accessibility and maintainability
- Let the interface feel tailored to its context rather than template-derived
